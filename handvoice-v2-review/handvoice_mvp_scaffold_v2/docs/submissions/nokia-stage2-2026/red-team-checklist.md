# Red-team review

Each reviewer records PASS, FAIL or NOT REVIEWED with a concrete correction.

| Reviewer lens | Attack question | Status |
|---|---|---|
| Professor | What is novel beyond TapTalk, and what evidence supports that boundary? | |
| Clinician/SLP | What construct do the instructions measure, and what may not be inferred? | |
| Community-care professional | What recurring decision does this change? | |
| Older-adult/accessibility | Can the task be understood and completed without avoidable burden? | |
| Engineer | What happens on poor light, poor audio, backgrounding, network failure and retry? | |
| Regulatory/privacy | Is any wording diagnostic, and where does identifiable media go? | |
| Nokia judge | Can strategy, integration, feasibility, impact and budget be understood in three minutes? | |
| Hostile judge | Why not clinician observation, TapTalk, or doing nothing? | |

Automatic fail: unsupported health outcome, invented interview/device/budget
result, live-demo dependency, missing human decision boundary, or unreadable
one-page export.
